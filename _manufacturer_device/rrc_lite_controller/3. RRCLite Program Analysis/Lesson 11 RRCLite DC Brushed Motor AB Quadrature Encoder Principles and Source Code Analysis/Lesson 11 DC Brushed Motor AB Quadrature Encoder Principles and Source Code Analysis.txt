1. Preface
DC gear motor is a high-torque, high-load motor available in 8V and 12V
variants, equipped with an encoder. This chapter focuses on controlling the
operation of the JGA27-310 DC gear motor (referred to as the 310 motor)
using an STM32 main control board.
Principle of AB quadrature encoder:
The rotary quadrature encoder is a digital IC (integrated circuit), so it uses
binary code. Square waves indicate digital signals in high (1) and low (0)
states.




The operation of the encoder under 5V (using 5V as an example) power supply
is illustrated in the above diagram, where the lines show low 0V (ground) and
high 5V states.
Velocity determination: The number of pulses per revolution of the disk
depends on the number of marks on its surface. This principle is the basic
definition of encoder resolution. Therefore, two output signals represent the
number of pulses in a specific time period. Using these values, the control
system can determine angular velocity, distance, and acceleration.
Direction determination:



                                          1
Two pins are placed sequentially. Thus, when simultaneously viewing the two
waveform lines, you will notice a slight offset. Output channels A and B are
specifically positioned on the disk at a certain distance, causing them to have a
90-degree phase offset on the waveform. The cycle is also divided into 4 equal
parts.




If the encoder moves forward (clockwise), line A is at the highest point (1), and
channel B is at the lowest point (0). This results in combinations like 1 and 0,
followed by 11, 00, and so on.
Starting from the same point in different directions (counterclockwise), you will
see that this line will first be at the highest point (11), followed by combinations
like 01, 00, and so on.

After powering on the STM32 control board, the motor rotates slowly for a
period of time, then rotates quickly for another period of time, cycling
repeatedly between these two operating states.

current motor. The current speed continuously approaches the preset speed
under the adjustment of the PID algorithm.

In this function, after the preset speed is limited, it is only assigned to the
speed variable member of the structure after being limited. It can be seen that

we only provide the target data.
Every 10000us, timer 7 overflows and enters an interrupt, where we perform
further operations on the set parameters. The detailed definition is as follows:




First, for each motor, the encoder_update function is used once to calculate
the incremental count recorded by the encoder (motor 1's encoder is
completed by timer 5 in encoder mode) since the last time it entered the
interrupt.
For motor 1, the parameter motors[0] provides the total count value of the last
time, the parameter __HAL_TIM_GET_COUNTER(&htim5) reads the current
count value of timer 5, and provides the count cycle of timer 7 (the time interval
between each timer interrupt trigger) 10000us=0.01s.
Assuming we know the count increment of one revolution of the motor (see the
manufacturer's data manual), and we know the count increment in 0.01s, we
can naturally know how many revolutions the motor has turned in 0.01s, and
thus know the current speed.
Then call the encoder_motor_control function to calculate the PID output
parameters based on the preset PID model, current speed, and target speed
